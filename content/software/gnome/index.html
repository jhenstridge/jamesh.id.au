James' Gnome Page

GNOME stands for GNU Network Object Model Environment. It aims to create a user friendly desktop environment based entirely on free software. For more information on GNOME itself, please see The GNOME website.

This page has some information about GNOME programs I have written. They are all under development, but are still useful and useable.

gnome-python

gnome-python is a set of bindings for the gnome libraries. It contains the PyGTK package, which is also written by me. It is available in source form from:
  • ftp.gnome.org:/pub/GNOME/sources/gnome-python/1.99/

GnoRPM

GnoRPM is a graphical front end to the RPM package management system. I wrote it and maintained it up til the end of 1999 (around the 0.9 release). It has not been ported Gnome 2.0 or RPM 4.1, so is not yet usable on the latest Linux distros. Some screenshots of earlier versions can be found here and here. You can find GnoRPM here:

  • ftp.gnome.org:/pub/GNOME/sources/gnorpm/

Libglade

Libglade is a library that performs a similar job to the C source output routines in the GLADE user interface builder. Whereas GLADE's output routines create C code that can then be compiled, libglade builds the interface from an XML file (GLADE's save format) at runtime. This way you can change the look of a program without needing to recompile.

Currently it supports all the widgets in current releases, together with support for keyboard accelerators and automatic signal connection.

It is available from:

  • ftp.gnome.org:/pub/GNOME/sources/libglade/2.0/

Gnome-VNC

Gnome-VNC is a GNOME viewer for Virtual Network Computing servers. VNC allows you to view and use the desktop of windows machines, and the display of the special Xvnc X server accross networks. The protocol is stateless, and multiple viewers can look at and interact with a single VNC display (good for presentations).

Currently gnome-vnc is only available from my site, as it contains some simple DES encryption routines, which are necessary to perform the password authentication. I replaced the original encryption routines that came with the VNC distribution with a modified version of the DES code in Samba (in fact I crippled the code further, so it has even fewer entry points, and is incapable of generating reverse key schedules used in decryption). So gnome-vnc is as legal as samba. Make up your own mind about whether you are allowed to use it.

It is available from:

  • ftp.daa.com.au:/pub/james/gnome
